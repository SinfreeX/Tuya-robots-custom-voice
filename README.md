# Robot Voice Tool

Локальный инструмент для сборки и отправки кастомного голосового пакета на Tuya/Ray-based робот-пылесосы вроде RP38.

## Запуск

Дважды кликнуть:

```bat
start_robot_voice.bat
```

Откроется Web UI: `http://127.0.0.1:8780/`.

## Что нужно пользователю

Для работы по локальной сети нужны:

- `Device ID` своего робота;
- `Local Key` своего робота;
- IP робота в LAN, который можно найти встроенным сканером.

`Device ID` и `Local Key` индивидуальны для каждого устройства. Их обычно получают через Tuya IoT Platform, привязав свой Smart Life/Tuya аккаунт к developer-проекту, или через совместимые инструменты вроде TinyTuya wizard. После получения этих данных облако Tuya для обычной отправки пакета не нужно: инструмент работает через локальный LAN-протокол.

Не публикуйте свой `config.json`: в нем хранится `Local Key`. Для GitHub в проекте есть `config.example.json`, а `.gitignore` исключает локальные секреты, пакеты и логи.

## Как пользоваться

1. Ввести `Device ID` и `Local Key`, нажать `Сохранить`.
2. Нажать `Сканировать сеть` и выбрать подтвержденного робота.
3. Указать папку с mp3-файлами. Имена должны быть как у штатного пакета: `C001.mp3`, `D001.mp3`, `E001.mp3`, `S001.mp3` и т.д.
4. Нажать `Собрать и отправить`.

Инструмент соберет `tar.gz`, посчитает MD5, выберет новый `languageId`, отдаст файл по HTTP и отправит DP35 роботу.

## Ротация languageId

`languageId` ротируется автоматически. Если отправлять тот же ID повторно, робот может ответить успехом без скачивания файла.
